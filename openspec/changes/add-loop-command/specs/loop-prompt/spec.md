## 循环提示系统

内置维护提示和 `loop.md` 自定义提示支持。

### 需求

- **MUST** 内置维护提示包含以下优先级任务：
  1. 继续会话中未完成的工作
  2. 检查当前分支的 PR 状态（CI 状态、审查评论、合并冲突）
  3. 无待办事项时执行清理（bug 修复、代码简化）
- **MUST** `loop.md` 查找顺序：`.claude/loop.md`（项目级）→ `~/.nanohermes/loop.md`（用户级）
- **MUST** 项目级 `loop.md` 优先于用户级
- **MUST** 两处都不存在时回退到内置维护提示
- **MUST** `loop.md` 内容超过 25,000 字节时截断
- **MUST** 每次循环迭代前重新加载 `loop.md`（支持运行时修改）
- **MUST** 当用户提供提示时，忽略 `loop.md` 和内置提示
- **SHOULD** 内置维护提示包含安全约束：
  - 不启动新的主动工作
  - 不可逆操作需要用户已授权
  - 禁止递归调度循环

### 内置维护提示文本

```
维护模式。按以下优先级处理：

1. 如果当前会话中有未完成的工作（之前的回复提到要做但还没完成的事），继续完成它。
2. 如果当前分支有活跃的 PR：
   - 检查 CI 状态，如果失败则拉取失败日志并诊断
   - 处理新的审查评论
   - 解决合并冲突
3. 如果以上都没有待办事项，执行清理任务：
   - 修复已知的 bug
   - 简化复杂的代码
   - 改进测试覆盖

注意：
- 不要启动新的主动工作，只处理已有上下文中的任务
- 不可逆操作（push、delete 等）只有在之前对话中已授权的情况下才执行
- 不要递归创建新的循环任务
```

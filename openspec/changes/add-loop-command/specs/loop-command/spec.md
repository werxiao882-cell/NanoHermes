## /loop 命令

用户通过 `/loop` 命令触发循环执行模式，支持 4 种调用方式。

### 需求

- **MUST** 支持 `/loop <interval> <prompt>` 格式：按固定间隔重复执行用户提示
- **MUST** 支持 `/loop <prompt>` 格式：按 AI 动态选择的间隔重复执行用户提示
- **MUST** 支持 `/loop <interval>` 格式：按固定间隔执行内置维护提示
- **MUST** 支持 `/loop` 格式：按动态间隔执行内置维护提示
- **MUST** 间隔单位支持 `s`（秒）、`m`（分钟）、`h`（小时）、`d`（天）
- **MUST** 最小间隔为 60 秒（1 分钟），小于此值自动提升
- **MUST** 最大间隔为 86400 秒（24 小时），超过此值拒绝
- **MUST** 同一时间只允许一个活跃循环，新循环需先停止旧循环
- **MUST** 创建循环时显示确认信息，包含间隔、提示摘要、循环 ID
- **MUST** 未识别的间隔格式给出友好错误提示

### 示例

```
/loop 5m 检查部署是否完成
→ 已创建循环 (ID: abc12345)，每 5 分钟执行：检查部署是否完成

/loop 检查 CI 是否通过
→ 已创建循环 (ID: def67890)，AI 将动态选择间隔执行：检查 CI 是否通过

/loop 10m
→ 已创建循环 (ID: ghi11111)，每 10 分钟执行维护任务

/loop
→ 已创建循环 (ID: jkl22222)，AI 将动态选择间隔执行维护任务
```

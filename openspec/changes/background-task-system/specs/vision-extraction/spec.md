## 新增需求

### 需求：视觉分析工具

系统应提供 `vision_analyze` 工具，使用多模态 LLM 能力分析图片内容。

#### 场景：用户请求图片分析
- **当** 用户调用 `vision_analyze(path="/path/to/image.png")`
- **则** 系统读取图片文件
- **则** 系统将图片发送给支持多模态的 LLM
- **则** 系统返回图片内容的详细描述

#### 场景：图片文件不存在
- **当** 用户调用 `vision_analyze`，传入无效路径
- **则** 系统返回错误："图片文件未找到：<path>"

#### 场景：文件不是图片
- **当** 用户对文本文件调用 `vision_analyze`
- **则** 系统从扩展名检测文件类型
- **则** 系统返回错误："文件不是支持的图片格式"

### 需求：视觉工具支持 OCR 模式

系统应提供专注于 OCR 的模式，从图片中提取文本。

#### 场景：用户请求 OCR 提取
- **当** 用户调用 `vision_analyze(path="...", ocr_only=true)`
- **则** 系统提示 LLM 提取所有可见文本
- **则** 系统以结构化格式返回提取的文本

#### 场景：图片不包含文本
- **当** 用户对不包含文本的图片调用 `vision_analyze`，`ocr_only=true`
- **则** 系统返回："图片中未检测到文本"

### 需求：视觉工具检查模型能力

系统应在尝试视觉分析前验证当前模型是否支持多模态输入。

#### 场景：模型支持视觉
- **当** 当前模型是 GPT-4o、Claude 3 或其他多模态模型
- **则** `check_fn` 返回 True
- **则** 工具可用

#### 场景：模型不支持视觉
- **当** 当前模型是纯文本模型（如 GPT-3.5、Claude 2）
- **则** `check_fn` 返回 False
- **则** 工具标记为不可用
- **则** 用户收到有用的错误："当前模型不支持图片分析。请使用多模态模型，如 GPT-4o 或 Claude 3。"

### 需求：视觉工具支持详细程度

系统应允许用户指定分析的详细程度。

#### 场景：低详细程度模式
- **当** 用户调用 `vision_analyze(path="...", detail="low")`
- **则** 系统返回简短摘要（1-2 句）
- **则** 系统使用较少的 token

#### 场景：高详细程度模式（默认）
- **当** 用户调用 `vision_analyze(path="...", detail="high")` 或省略 detail 参数
- **则** 系统返回全面分析（段落）
- **则** 系统描述对象、颜色、布局、文本和上下文

### 需求：视觉工具处理大图片

系统应调整或压缩大图片，以适应模型 token 限制。

#### 场景：图片超过大小限制
- **当** 图片文件大于 20MB
- **则** 系统调整图片大小以适应限制
- **则** 系统保持宽高比
- **则** 系统在 DEBUG 级别记录调整操作

#### 场景：图片在大小限制内
- **当** 图片文件小于 20MB
- **则** 系统不修改直接发送图片

### 需求：视觉工具可发现

系统应将 `vision_analyze` 注册为延迟加载工具，可通过 `search_tools` 发现。

#### 场景：用户搜索视觉能力
- **当** 用户调用 `search_tools(query="analyze image")`
- **则** 系统在结果中返回 `vision_analyze`
- **则** 系统包含描述："使用多模态 AI 分析图片内容并提取文本"

#### 场景：工具加载
- **当** `vision_analyze` 被发现
- **则** 系统将工具加载到活动工具集
- **则** 工具立即可用

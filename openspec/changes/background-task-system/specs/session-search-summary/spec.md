## 新增需求

### 需求：会话搜索结果摘要

系统应为会话搜索结果提供可选的 LLM 驱动摘要功能。

#### 场景：用户请求带摘要的搜索
- **当** 用户调用 `session_search(query="...", summarize=true)`
- **则** 系统照常执行 FTS5 搜索
- **则** 系统将搜索结果发送给 LLM 生成摘要
- **则** 系统返回原始结果和生成的摘要

#### 场景：用户请求不带摘要的搜索（默认）
- **当** 用户调用 `session_search(query="...")`，不带 `summarize` 参数
- **则** 系统执行 FTS5 搜索
- **则** 系统仅返回原始结果（当前行为）
- **则** 系统不调用 LLM

#### 场景：摘要生成优雅失败
- **当** 摘要生成的 LLM 调用失败或超时
- **则** 系统在 WARNING 级别记录警告
- **则** 系统返回不带摘要的原始搜索结果
- **则** 系统包含说明："摘要生成失败，显示原始结果"

### 需求：搜索摘要尊重上下文

系统应生成考虑用户当前查询意图的摘要。

#### 场景：查询特定的摘要
- **当** 用户使用 `summarize=true` 搜索 "authentication bug"
- **则** 系统提示 LLM："在查找身份验证 bug 的上下文中总结这些搜索结果"
- **则** 摘要突出显示相关的匹配和模式
- **则** 摘要省略不相关的信息

#### 场景：摘要包含可操作的见解
- **当** 搜索结果包含多个相关会话
- **则** 摘要识别共同主题
- **则** 摘要建议哪些会话最相关
- **则** 摘要可能推荐下一步操作

### 需求：搜索摘要有 token 限制

系统应限制摘要长度，控制成本并避免信息过载。

#### 场景：强制执行摘要 token 限制
- **当** LLM 生成摘要
- **则** 系统设置 `max_tokens=500` 用于摘要生成
- **则** 如果摘要超过限制，系统截断
- **则** 如果截断，系统追加 "...（摘要已截断）"

#### 场景：搜索结果较少
- **当** 搜索仅返回 2-3 个结果
- **则** 摘要简洁（2-3 句）
- **则** 系统不必要地填充摘要

### 需求：搜索摘要可配置

系统应允许用户通过配置启用/禁用搜索摘要。

#### 场景：搜索摘要被禁用
- **当** `background_tasks.session_search_summary.enabled` 设置为 `false`
- **则** `summarize=true` 参数被忽略
- **则** 系统仅返回原始结果
- **则** 系统记录信息："搜索摘要已禁用"

#### 场景：搜索摘要被启用（默认）
- **当** `background_tasks.session_search_summary.enabled` 未设置或设置为 `true`
- **则** `summarize=true` 参数触发 LLM 摘要生成

### 需求：搜索摘要缓存结果

系统应缓存最近的搜索摘要，避免重复的 LLM 调用。

#### 场景：相同的搜索查询
- **当** 用户在 5 分钟内搜索相同的查询
- **则** 系统返回缓存的摘要
- **则** 系统不再次调用 LLM
- **则** 系统在 DEBUG 级别记录缓存命中

#### 场景：缓存过期
- **当** 距离上次搜索已过 5 分钟
- **则** 缓存条目过期
- **则** 下次搜索生成新摘要

#### 场景：缓存大小限制
- **当** 缓存包含超过 20 个条目
- **则** 系统驱逐最旧的条目（LRU）
- **则** 系统保持缓存大小在 20 或更少

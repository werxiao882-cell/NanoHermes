## 一、测试概述

### 1.1 测试目标

对 NanoHermes AI Agent 系统进行**真实环境下的端到端功能验证**，确保所有核心模块在实际运行中按预期工作。

### 1.2 测试范围

| 模块 | 测试重点 | 用例数 |
|------|---------|--------|
| session-storage | 会话创建、消息存储、持久化、并发 | 7 |
| tool-runtime | 工具调用、执行、错误处理 | 12 |
| memory-system | 记忆注入、更新、持久化 | 5 |
| provider-runtime | LLM API 调用、流式输出、Token 计数、回退链 | 6 |
| unified-config-system | 配置加载、优先级链、验证、多提供商 | 7 |
| system-prompt-assembly | 三层架构、缓存控制、威胁检测 | 8 |
| tool-search | 延迟加载、BM25/Regex 搜索、Auto 模式 | 6 |
| conversation-loop | 对话循环、上下文保持、错误恢复、事件总线 | 10 |
| multi-agent-delegation | leaf/orchestrator 角色、并发控制 | 5 |
| skill-system | 技能列表、加载、注入、创建、更新 | 6 |
| context-compression | 压缩触发、摘要生成、头尾保护 | 5 |

| add-mcp-server-support | MCP 模块加载、工具桥接、三种传输模式 | 6 |

| auxiliary | 后台任务、记忆刷写、凭证复用、错误隔离 | 4 |
| **总计** | **15 个核心模块** | **131** |

### 1.3 测试环境要求

```bash
# 必需环境
- Python >= 3.11（推荐 3.12）
- conda 环境 py312（或等效虚拟环境）
- WSL2 / Linux 环境
- 有效的 LLM API Key（DashScope/OpenAI/Anthropic）

# 项目依赖
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple \
    openai anthropic pyyaml pydantic python-dotenv \
    rich prompt_toolkit better-sqlite3 \
    pytest pytest-asyncio pytest-cov
```

## 二、测试准备

### 2.1 环境配置

```bash
# 1. 激活 conda 环境
eval "$($HOME/miniconda3/bin/conda shell.bash hook)"
conda activate py312

# 2. 进入项目目录
cd /mnt/d/code/NanoHermes

# 3. 配置 .env 文件（如不存在）
cat > .env << 'EOF'
DASHSCOPE_API_KEY=your-api-key-here
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
MODEL_NAME=qwen3.6-plus
EOF

# 4. 清理旧测试数据（可选）
rm -rf ~/.nanohermes/sessions/*
rm -f ~/.nanohermes/sessions.db
rm -rf ~/.nanohermes/memory/*
```

### 2.2 测试数据准备

| 数据类型 | 路径 | 说明 |
|---------|------|------|
| 会话存储 | `~/.nanohermes/sessions/` | JSONL 文件，每会话一个 |

| 记忆文件 | `~/.nanohermes/memory/MEMORY.md` | Agent 记忆 |
| 用户画像 | `~/.nanohermes/memory/USER.md` | 用户偏好 |
| MCP 配置 | `~/.nanohermes/mcp_servers.json` | 外部 MCP 服务 |

## 三、测试执行方法

### 3.1 核心测试方式：PTY 交互测试

**原理**: 启动 NanoHermes 主程序，通过 PTY（伪终端）进行真实对话交互，观察 AI 行为和功能响应。

```bash
# 启动方式
python -m src.main              # 正常模式
python -m src.main --debug      # 调试模式（输出完整请求/响应 JSON）
python -m src.main --resume     # 恢复最近会话
```

### 3.2 测试执行步骤

```bash

# - 配置加载成功
# - 工具注册完成
# - 记忆加载完成
# - 进入交互模式
```

**检查点**:
- [ ] 无启动错误

## 七、已知问题与注意事项

### 7.1 已知问题

| 问题 | 现象 | 影响 | 状态 |
|------|------|------|------|
| MEMORY.md 重复条目 | 相同信息多次写入 | 低（功能正常，文件冗余） | 待修复 |
| search_files 过度搜索 | AI 连续调用多次搜索 | 低（效率可优化） | 待优化 |

### 7.2 测试注意事项

1. **API 成本**: 真实 API 测试会产生费用，控制测试轮次
2. **速率限制**: 注意 API 调用频率，避免触发限流
3. **数据清理**: 测试前备份重要数据，测试后清理测试会话
4. **环境隔离**: 使用独立的测试目录，避免污染生产环境
5. **超时处理**: 长时间无响应时检查网络连接和 API Key

### 7.3 测试失败排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 启动失败 | 依赖缺失 | `pip install -r requirements.txt` |
| API 调用失败 | Key 无效/网络问题 | 检查 .env 配置 |
| 工具调用超时 | 命令执行慢 | 增加 timeout 参数 |
| 记忆未保存 | 权限问题 | 检查 ~/.nanohermes/ 权限 |
| 会话未存储 | 数据库锁 | 关闭其他进程，删除 .db-wal |
